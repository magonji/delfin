❓ How to import a bank CSV

Delfin imports any bank's CSV export. The first time you import from a bank you tell Delfin which column is which; it saves that as a profile so next time the same bank is recognised automatically and you go straight to the preview.

Steps

  1. Select the account the statement belongs to, then choose the CSV file.
  2. This mapping screen appears (only for unknown files). Delfin pre-fills its best guesses — check them against the preview table at the bottom and fix anything wrong.
  3. Optionally type a profile name (e.g. your bank's name) and click Save profile so future imports skip this screen.
  4. Click Continue to see the full preview, where you can review/assign payees and categories, then Import.

What each field means

  • Delimiter — the character separating columns: comma , (UK/US) or semicolon ; (common in Spain and the rest of Europe).
  • Encoding — how the file's text is stored. Leave on Auto-detect; switch to Windows-1252 / ISO-8859-1 only if accents look wrong (e.g. NÃ³mina).
  • Decimal — the decimal symbol in amounts: point 1234.56 or comma 1.234,56.
  • Date format — how dates are written in the file, e.g. DD/MM/YYYY.
  • Date column — the column holding each movement's date.
  • Amount type — Single column if one column holds the amount with a sign (negative = expense), or Debit / Credit if the bank uses two separate columns (charges in one, deposits in the other).
  • Sign — usually As-is. Choose Flip if your bank lists expenses as positive numbers.
  • Description column(s) — the text that names each movement (shop, payee…). Hold Ctrl/Cmd to combine several columns.
  • Note column (optional) — an extra column kept as the transaction's note (a reference or comment). Leave on — none — if you don't need it.
  • Saved profile — pick a previously saved profile to reload all of the above at once.

Good to know

  • Movements are shown oldest → newest in the preview.
  • Duplicates already in Delfin are detected and skipped, so re-importing an overlapping statement is safe.
  • When a movement matches a known payee, its usual category is filled in for you automatically.

♻️ Restore Database

Restore a Delfin database backup — a .db file from the Backup tool or the automatic daily backups. This replaces all your current data with the file's contents.

🛡️ A safety backup of your current database is created automatically first.

⏳ Restoring…

🔑 Change password

The data key is re-wrapped with the new password — the database itself is not re-encrypted, and existing backups stay valid (they're tied to the data key, not the password). Your recovery code is unchanged.

🛠️ Maintenance

Once a day the app refreshes exchange rates, recalculates balances and payee statistics, and backs up the database to a second disk. The backup is only written when your data has actually changed (exchange-rate updates alone don't count).
